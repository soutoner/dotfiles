# Copilot Instructions

## General Principles
- Be concise and direct
- Prefer editing existing files over creating new ones
- Use the Task tool for codebase exploration
- Always use TodoWrite to plan and track multi-step tasks

## Git & Commits
- **Do NOT auto-commit** unless explicitly confirmed by the user
- Always show the user what will be committed before creating a commit
- Provide a summary of changes and ask for confirmation
- Only commit when the user explicitly requests it

## Code Style
- Follow existing patterns in the codebase
- Maintain XDG Base Directory Specification compliance when tool supports it
- Keep shell scripts and configs consistent with existing style
- Before installing a tool or plugin verigy if it is already installed

## Documentation
- Update README.md when adding features or changing provisioning
- Keep documentation accurate, current and concise
- Avoid creating new markdown files unless necessary
- Do not add repository structure section
